# 📖 What is the Double Ratchet Algorithm?

The **Double Ratchet Algorithm** is the core cryptographic mechanism used in secure messaging applications such as Signal.

It combines:

1. **Diffie-Hellman Ratchet**

   * Continuously generates new shared secrets.
   * Provides post-compromise security.

2. **Symmetric-Key Ratchet**

   * Derives a new encryption key for every message.
   * Provides forward secrecy.

Together, these ratchets ensure that compromising a current key does not expose past or future messages.

# 🚀 Quick Start

## 1. Import the Module

```python
from double_ratchet import DoubleRatchet
```

---

## 2. Establish a Shared Secret

The Double Ratchet requires a secure 32-byte shared secret.

Typically this is generated by an X3DH key exchange.

```python
shared_secret = x3dh_shared_secret
```

---

## 3. Initialize Alice

```python
alice = DoubleRatchet(
    shared_secret,
    bob_public_key,
    is_bob=False
)
```

---

## 4. Initialize Bob

```python
bob = DoubleRatchet(
    shared_secret,
    bob_private_key,
    is_bob=True
)
```

---

# 🔒 Encrypting Messages

```python
header, ciphertext = alice.encrypt(
    b"Hello Bob!"
)
```

The encryption process:

1. Advances the sending chain.
2. Generates a fresh message key.
3. Encrypts using AES-256-GCM.
4. Produces a ratchet header.

---

# 🔓 Decrypting Messages

```python
plaintext = bob.decrypt(
    header,
    ciphertext
)

print(plaintext.decode())
```

Output:

```text
Hello Bob!
```

# 💾 Session Persistence

Save the complete ratchet state:

```python
session_data = ratchet.to_dict()
```

Restore later:

```python
ratchet = DoubleRatchet.from_dict(
    session_data
)
```

This allows sessions to survive:

* Server restarts
* Database storage
* User reconnects

---

# 🔄 Out-of-Order Message Handling

The implementation supports delayed or reordered messages.

Example:

```python
msgA = alice.encrypt(b"A")
msgB = alice.encrypt(b"B")

bob.decrypt(*msgB)
bob.decrypt(*msgA)
```

Messages are successfully decrypted even when delivered out of order.

---

# 🛡 Tamper Detection

AES-GCM authentication ensures message integrity.

Example:

```python
tampered = bytearray(ciphertext)
tampered[-1] ^= 0xFF

bob.decrypt(header, bytes(tampered))
```

Result:

```text
DecryptionError
```

Any modification to:

* Ciphertext
* Authentication Tag
* Header Data

will be detected and rejected.

# 🔐 Security Properties

## Forward Secrecy

Compromise of a current key does not reveal past messages.

## Post-Compromise Security

Once a new DH ratchet step occurs, future messages become secure again even after a compromise.

## Message Integrity

AES-GCM authentication prevents undetected modification.

## Replay Protection

Each message uses a unique ratchet-derived message key.
